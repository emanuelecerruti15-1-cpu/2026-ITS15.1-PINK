2. CREAZIONE MODEL PIATTO
- Creato il package model.
- Creato il file Piatto.java.
- Definiti gli attributi base del piatto:
  - id
  - nome
  - prezzo
  - disponibile
- Creato il costruttore vuoto.
- Creato il costruttore con parametri.
- Creati getter e setter.

- Verificato che Piatto.java compila e funziona correttamente.

- Creato il package service.
- Creato il file PiattoService.java.

- Creato un sistema automatico per generare gli id dei piatti.
- Inseriti piatti iniziali di esempio:
  - Pizza Margherita
  - Spaghetti Carbonara
  - Tiramisù
- Implementata la funzione per recuperare tutti i piatti.
- Implementata la funzione per recuperare un piatto tramite id.
- Implementata la funzione per creare un nuovo piatto.
- Implementata la funzione per modificare un piatto esistente.
- Implementata la funzione per eliminare un piatto.
- Implementata la funzione per cambiare la disponibilità di un piatto.
- Implementate validazioni base:
  - nome non vuoto
  - prezzo maggiore di zero

- Creato il file PiattoController.java.
- Collegato PiattoController a PiattoService.
- Esposto l’endpoint base:
  - /api/piatti
- Implementata API per ottenere tutti i piatti:
  - GET /api/piatti
- Implementata API per ottenere un singolo piatto:
  - GET /api/piatti/{id}
- Implementata API per creare un piatto:
  - POST /api/piatti
- Implementata API per modificare un piatto:
  - PUT /api/piatti/{id}
- Implementata API per eliminare un piatto:
  - DELETE /api/piatti/{id}
- Configurato @CrossOrigin per permettere le chiamate dal frontend.
- Corretto l’uso di @PathVariable("id").
